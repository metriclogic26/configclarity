Docker Healthcheck

A Docker healthcheck is a command defined in a container's configuration that Docker runs periodically to determine if the container is functioning correctly — allowing orchestrators to restart failed containers and delay dependent services.
DockerHealthcheckOrchestrationContainerDevOps

A Docker healthcheck defines a test command that Docker runs inside the container at a regular interval. If the command exits with a non-zero status code, Docker marks the container as unhealthy. Container orchestrators (Docker Compose, Swarm, Kubernetes) can then take action — restarting the container, stopping dependent services from starting, or routing traffic away from unhealthy instances.

Without a healthcheck, Docker considers a container healthy as soon as it starts. The container may be running but the application inside may not be ready to serve traffic — the web server may still be starting up, the database may be running its initial migration, or a connection pool may not yet be established.

Healthcheck in docker-compose.yml

healthcheck:
  test: ["CMD", "curl", "-f", "http://localhost:8080/health"]
  interval: 30s
  timeout: 10s
  retries: 3
  start_period: 40s

start_period is the grace period before health failures count against the retry limit — essential for services with slow startup like Java applications or services running database migrations on boot.

Frequently Asked Questions

What happens if a container has no healthcheck?
Docker reports the container status as 'running' regardless of whether the application is functioning. Dependent services start immediately without waiting for the container to be ready. Orchestrators cannot detect application failures — only process crashes.
What is a good healthcheck command for a web service?
For HTTP services: ["CMD", "curl", "-f", "http://localhost:PORT/health"]. For services without curl: ["CMD-SHELL", "wget -q --spider http://localhost:PORT/health || exit 1"]. For databases: use the database's own health tool — ["CMD", "pg_isready", "-U", "postgres"] for PostgreSQL.
How do I check if my containers have healthchecks configured?
Paste your docker-compose.yml into ConfigClarity's Docker Auditor. It flags every service missing a healthcheck definition and generates a suggested healthcheck command based on the detected service type.
