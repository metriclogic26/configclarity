Docker Compose Linter

Free online validator — paste your compose file, get instant results

What this linter checks

Security checks
Hardcoded passwords, API keys, and tokens in environment blocks. 0.0.0.0 port bindings exposing services to the internet.
Reliability checks
Missing healthchecks on services used with depends_on: service_healthy. Port collisions across services.
Image checks
Services using :latest tag (unpredictable builds). Digest-pinned images (rewarded). Docker Hub tag verification.
GPU checks
Deprecated runtime: nvidia syntax. Missing NVIDIA_VISIBLE_DEVICES. Old-style /dev/nvidia0 device bindings.

Common issues this catches

1. Hardcoded database passwords

❌ Flagged as CRITICAL 
environment:
  - POSTGRES_PASSWORD=mysecretpassword  # literal value

2. Port exposure via 0.0.0.0 binding

❌ Flagged as CRITICAL — Redis publicly accessible 
ports:
  - "6379:6379"  # binds to all interfaces

3. :latest image tags

❌ Flagged — unpredictable, breaks reproducibility 
image: redis:latest

4. Missing healthcheck on dependency

❌ Flagged — depends_on: service_healthy requires a healthcheck 
depends_on:
  db:
    condition: service_healthy
# db has no healthcheck: block defined

Frequently Asked Questions

Is this the same as docker compose config?

docker compose config validates YAML syntax and resolves variables. This linter goes further: it checks for security issues (hardcoded secrets, port exposure), reliability issues (missing healthchecks, port collisions), and image tag hygiene. Think of it as a security-focused layer on top of basic YAML validation.

Does it check version compatibility?

The linter parses any valid docker-compose.yml regardless of version field. Note that the version: field is deprecated in Compose V2 — it's ignored by modern Docker installations.

Can I lint multiple compose files?

Paste one file at a time. For multiple files, run the linter once for each. A future feature will support override file merging.

Does it validate environment variable references?

Yes — paste your .env file in the optional second textarea and the linter shows which variables are referenced in the compose file but missing from .env, and which .env variables are unused by any service.
